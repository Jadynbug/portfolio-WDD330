  3. Translation

    Translation lets you make a change in color/design less sudden

    You need to set a before and after for each property translated. like "a" vs "a:hover"

    transition-property declares what properties can be translated

    transistion-duration declares how long it should take to get from pre to post

    transition-timing-function lets you declare is you want it fast at first, slow, slow at first, ect.

    transistion-delay lets you add a delay timer before the transitions happen. A negative delay will let it start in the middle

    If you synce the timming function and the property declaration you can have different things change at different paces

  4. Animations

    animations use keyframes to set up many transitions in a sequence.

                        @keyframes moveRight { 
                        from {
                        transform: translateX(-50%);
                        }
                        to {
                        transform: translateX(50%);
                        }
                    }
                    
                    @keyframes appearDisappear { 
                        0%, 100% {
                        opacity: 0;
                        }
                        20%, 80% {
                        opacity: 1;
                        }
                    }
                    
                    @keyframes bgMove {
                        100% {
                        background-position: 120% 0;
                        }
                    }

    you make keyframes and then assign things to use them:
    animation-name: appearDisappear;

    animation-duration declares how long an animation has to go from 0% - 100%

    animation-timing-function declares what speed the animation is executed. Fast, slow, a mix?

    steps() lets you move multiple pics through the background to make it look like it's moving???

    animation-iteration-count declares how many times an animation will play through. 1? infinite?

    animation-direction what if you want to go in reverse? try this one.

    animation-delay lets start late or in the middle.

    animation-fill-mode declares what happens between end and restart

    animation-play-state declare if the animation is playing or paused

  5. Canvas

    You set the width and height on the element because that defines your grid.

    You draw on the canvas using a JS API

    First set up the context

                        var canvas = document.getElementById("myCanvas");
                    var context = canvas.getContext("2d");

    Then you set up your brush using strokestyle or fillstyle

                        context.strokeStyle = "red";

    If you want a shape to have a outline and a fill you define both

    context.strokeStyle = "red";
                    context.fillStyle = "rgba(0, 0, 255, 0.5)";

    To draw you use strokeshape and fillshape. In this instance, strokerect and fillrect

                        context.fillRect(10, 10, 100, 100);   
                    context.strokeRect(10, 10, 100, 100);

    Drawing shapes use the grid to draw. It takes the x,y and then the sides a,b. So, strokeshape(x,y, a,b

    the canvas's top left is 0,0 and it's bottom right is 200,200 (or whatever width/height you set)
