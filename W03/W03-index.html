Week 03 Notes

  1. JS objects look like Python dictionaries.

                        let object = {
                        key: value,
                        key: value
                    }

  2. You can give objects functions.

                        let object = {
                        key: value,
                        key: value,
    
                        function() {do thing;}
                    }
    Then call those functions
    object.function();

  3. Try to avoid using "this" instead use the name of the object. JS "this" is not bound like other languages.

    Exceptions being: 

                        let object_1 = {name: "John"};
                    let object_2 = {name: "Steve"};

                    function sayHi() {alert(this.name);}

                    object_1.f = sayHi;
                    object_2.f = sayHi;
    By defining the function outside of the objects it can now work with either object due to JS's ability to use context.

    However, this means that the function will not work unless paired with an object. On it's own it will error.

  4. For (key in object) is a usefull loop to know

  5. object.keys returns all keys, object.values returns all values, and object.entries returns key:value pairs.

  6. Cloning objects means that changing the original changes the clone.

  7. A JSON object comes in this form: 

                        object = '{"key":"value","key":"value","key":"value"}'
    JSON.parse(object) truns a JSON string into a JS object: 
                        object = {
                        key: 'value',
                        key: 'value',
                        key: value
                    }
    JSON.stringify(object) turns a JS object into a JSON string
